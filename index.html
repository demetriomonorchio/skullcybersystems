ICT - 0 to Hero

Ideato strutturato e pensato per MrVinnie da Skull Cyber Systems

Course blueprint

Un percorso pratico pensato per chi parte da zero e vuole diventare operativo in Linux, virtualizzazione, networking, cloud e security attraverso laboratori ripetibili e progetti reali. Il corso combina lezioni teoriche brevi, esercitazioni guidate su VM/ambienti emulati e challenge di sicurezza per consolidare competenze spendibili sul mercato.

Target, prerequisiti, risultati
  • • Target: principianti motivati, junior sysadmin, studenti IT.
  • • Prerequisiti: PC con 16 GB RAM consigliati; curiosità e voglia di smanettare.
  • • Risultati attesi: saper installare e amministrare una VM Linux, creare topologie di rete emulate, deployare risorse cloud base, applicare hardening e svolgere semplici test di sicurezza; portfolio con 3 mini-progetti.
Programma completo
Modulo 1
Linux base

Comandi shell, utenti, permessi, filesystem

Obiettivo: fondamenti operativi su VM Linux. Durata: 2 settimane.

Lab: LabEx/GitHub + OverTheWire Bandit
Modulo 2
Servizi

Systemd, package, log, backup

Obiettivo: amministrazione e servizi con logging e backup. Durata: 1 settimana.

Lab: VM + deploy runbook backup/restore
Modulo 3
Virtualizzazione

VM, snapshot, rete virtuale, resource control

Obiettivo: creare VM e rendere i test rollbackable. Durata: 2 settimane.

Lab: VMware Workstation/ESXi + snapshot/rollback
Modulo 4
Networking

OSI, IP, subnetting, VLAN, routing base

Obiettivo: networking pratico con topologie e subnetting. Durata: 2 settimane.

Lab: EVE-NG/PNETLab/GNS3 + topologia funzionante
Modulo 5
Cloud

Cloud fundamentals: IaaS vs PaaS

Obiettivo: deployare VM su cloud e mappare concetti da on-prem a cloud. Durata: 2 settimane.

Lab: AWS/GCP/Azure free tier + security groups + deploy
Modulo 6
Security pratico

Hardening, patching, firewall, pentest etico

Obiettivo: sicurezza in lab isolato. Durata: 2 settimane.

Lab: Kali, Metasploitable, OverTheWire + checklist hardening
Modulo 7
Capstone

Capstone project: integrare web app, rete, cloud, monitor, hardening

Obiettivo: progetto finale completo con documentazione. Durata: 2 settimane.

Output: portfolio progetto completo + runbook
Materiali e piattaforme
  • • OverTheWire: wargames (progressione guidata). overthewire.org
  • • EVE-NG: emulatore di rete multi-vendor. EVE-NG
  • • PNETLab: alternativa con lab condivisi. pnetlab.com
  • • Coursera / LabEx / GitHub practice challenges: hands-on su Linux e amministrazione.
  • • Documentazione ufficiale (Ubuntu/CentOS/VMware/AWS/Azure) per riferimenti tecnici.
Metodologia didattica e valutazione
  • • Formato: mix asincrono (video + slide) e sincrono (lab guidati in diretta).
  • • Learning-by-doing: ogni lezione teorica ha un lab obbligatorio; uso intensivo di snapshot per rollback.
  • • Valutazione: quiz settimanali, esercizi pratici con rubriche, progetto finale valutato (codice, configurazioni, runbook).
  • • Certificazione interna: attestato di completamento + badge per moduli (utile per LinkedIn).
Struttura del corso

Corso 8 settimane, 1 lezione a settimana da 2 ore (16 ore totali). Ogni sessione combina 20–30 min di teoria, 80–90 min di lab guidato e 10–20 min di debrief + compiti. Obiettivo: arrivare a un capstone pratico che dimostri competenze Linux → virtualizzazione → networking → cloud → security.

Calendario dettagliato per sessione
Sessione 1
Intro & setup — installare VM host e creare prima VM Ubuntu.
Deliverable: screenshot VM + `uname -a` e `lsblk`.
Sessione 2
Comandi shell — permessi, utenti, editor nano/vim.
Deliverable: log comandi Bandit + riflessione.
Sessione 3
Processi & systemd — logging, journalctl e recovery via snapshot.
Deliverable: runbook di 6 passi per recovery servizio.
Sessione 4
Virtualizzazione — snapshot, clone, NAT vs bridged, resource allocation.
Deliverable: video/screencast del rollback.
Sessione 5
Networking — IP, subnetting, NAT, DNS, DHCP, OSI.
Deliverable: esercizi subnetting risolti.
Sessione 6
EVE-NG/PNETLab — importare topologie, collegare VM e verificare routing.
Deliverable: topologia esportata + screenshot console.
Sessione 7
Cloud basics — mapping VM → cloud, security groups e deploy.
Deliverable: screenshot console cloud + regola firewall applicata.
Sessione 8
Security capstone — hardening, backup, logging, revisione progetto finale.
Deliverable: report vulnerabilità + checklist hardening.
Requisiti tecnici, licenze e sicurezza legale
  • • Host consigliato: laptop con almeno 16 GB RAM; per lab condivisi server con 64+ GB RAM.
  • • Licenze: verificare EVE-NG/PNETLab e eventuali immagini vendor; usare immagini open o accordi per uso commerciale.
  • • Sicurezza: lab di pentest sempre isolati dalla rete pubblica; includere disclaimer legale e policy di uso etico nel materiale del corso.
Materiale didattico pronto che posso preparare
  • • Script passo-passo per ogni sessione (comandi, file di configurazione, snapshot points).
  • • Esercizi Bandit selezionati per sessioni 2 e 8.
  • • Template runbook per deploy, rollback e hardening.
  • • Demo video 10–15 min per la landing page.
